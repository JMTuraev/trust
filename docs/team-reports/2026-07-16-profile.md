## 2. OBUNA HAYOT SIKLI XARITASI

### 2.1 Ma'lumot modeli (008_profile_lifecycle.sql)
- `profiles.premium_until timestamptz` — pullik obuna tugashi (null = hech qachon sotib olinmagan).
- Trial uchun ustun YO'Q: `trial_ends_at = created_at + 7 kun` (API hisoblaydi).
- `profiles.deleted_at` — soft-delete (Play/App Store talabi), OTP qayta kirishda tiklanadi.

### 2.2 Holat mashinasi (endi yagona manba: `src/lib/subscription.js`)
```
premium : premium_until > now            (qat'iy >)
trial   : now < created_at + 7 kun       (qat'iy <)
expired : aks holda
```
- **Chegara:** aynan `trial_ends_at` yoki `premium_until` daqiqasida user allaqachon expired/nopremium —
  ikkala taqqoslash ham qat'iy. Testlangan (exactly-at-boundary → expired).
- **Timezone:** Postgres `timestamptz` ISO-UTC qaytaradi, `new Date()` epoch-millis bilan solishtiradi —
  server TZ'iga bog'liq emas. `+00:00` format parse testi PASS.
- **Qiziq holat (to'g'ri ishlaydi):** premium tugagan, lekin trial hali tugamagan (3-kunlik user premium
  refund bo'lsa) → `trial` qaytadi.
- **Grace:** `SUB_GRACE_DAYS` env (default 0 — joriy xatti-harakat o'zgarmagan). >0 qilinsa expired user
  yana N kun yoza oladi (`can_write=true`) — Play Billing kechikishlari uchun. Faqat `can_write`ga ta'sir
  qiladi, `status`ga EMAS (UI baribir "tugagan"ni ko'rsatadi).

## 4. AUTH XAVFSIZLIK AUDITI (src/routes/auth.js + services/otp.js + middleware/auth.js)

### Yaxshi (production darajasida)
- **JWT:** HS256 `algorithms:['HS256']` pin (alg-confusion yopiq), `audience:'authenticated'` tekshiriladi.
  Claims: `sub` (user id), `phone`, `role`, `aud`, `iat`, `exp` (7 kun). Secret: `APP_JWT_SECRET`
  (Render generateValue — kuchli).
- **OTP saqlash:** sha256 hash (ochiq kod DB'da yo'q), TTL `OTP_TTL_SECONDS` (300s), muvaffaqiyatda kod
  O'CHIRILADI (qayta ishlatib bo'lmaydi), eski kodlar yangi yuborishda o'chiriladi.
- **Brute-force:** kod boshiga `OTP_MAX_ATTEMPTS` (5) DB'da; 5 xonali kod → to'g'ri topish ehtimoli
  5/90000 ≈ 0.006%. verify-otp IP 10/min.
- **Toll-fraud:** send-otp IP 3/min + per-telefon 60s dedup + butun-servis soatlik cap
  (`SMS_GLOBAL_HOURLY_CAP`, haqiqiy yuborishdan OLDIN sanaladi — login-DoS yo'q).
- **trust proxy = 1** (aniq hop) — X-Forwarded-For spoofing bilan per-IP limitni chetlab bo'lmaydi.
- 5xx xabarlari mijozga oshkor qilinmaydi (info disclosure yopiq).
- Xalqaro raqamlar: Supabase OTP (o'z rate-limitlari bilan), token baribir bizniki.

### Kamchiliklar / xavflar
| # | Topilma | Jiddiylik | Tavsiya |
|---|---|---|---|
| A1 | **Refresh token yo'q** (`refresh_token: null`) — 7 kunda majburiy qayta-OTP (SMS xarajati + friction) | O'rta | Keyingi bosqich: refresh token jadvali yoki exp'ni 30 kunga uzaytirish (PIN darvozasi bor) |
| A2 | **O'chirilgan (deleted_at) akkaunt tokeni amal qiladi** — requireAuth DB'ni tekshirmaydi. O'g'irlangan token bilan "o'chirilgan" user API'ni ishlataveradi (7 kungacha) | O'rta | `requireActiveSub` deleted_at'ni yozish endpointlarida yopadi (qo'shimcha foyda). To'liq yechim: profiles.token_min_iat ustuni + middleware tekshiruvi — alohida qaror |
| A3 | OTP hash tuzsiz sha256 — 90k kombinatsiya rainbow trivial, LEKIN kod 5 daqiqa yashaydi va DB service-role ortida | Past | Xohlasa HMAC(APP_JWT_SECRET) — shart emas |

## 5. ONBOARDING AUDITI

Oqim: `boot` (splash, token bor-yo'qligi) → `welcome` → `phone` → `otp` → `pin(set→confirm)` → `app`.
Qayta kirish: `boot` → (`Api.me()` ok) → `pin(check)` → `app`; 401 → token o'chadi → `welcome`;
tarmoq xatosi → token SAQLANIB app'ga kiradi (hydrate keyin uradi) — to'g'ri ajratilgan.

- **Til tanlash:** onboardingda yo'q (prototipda ham yo'q) — default `uz`, keyin Profil → Til. Prototip
  1:1 talabiga mos; store `trust_lang`ni init'da o'qiydi. OK.

- **Login bilan yakunlanadimi:** ha — verify-otp → token secure storage'ga, PIN o'rnatiladi, `app`. OK.
- **G'alati holat (dead-end EMAS):** PIN-set ekranida "orqaga" → OTP ekraniga qaytadi, lekin kod allaqachon
  ishlatilgan (user allaqachon LOGIN BO'LGAN). User eski kodni kiritsa "Kod topilmadi" oladi; telefon
  ekranidan qayta SMS so'rasa yangi sessiya ochiladi — chiqib ketolmaydi, lekin chalkash. Prototip ham
  shunday (1376-1381). Taklif (ixtiyoriy, keyin): pin 'set' rejimida orqaga tugmasini yashirish.
- **OTP qayta yuborish tugmasi yo'q** — prototipda ham yo'q; user orqaga → «Davom etish» bilan qayta
  yuboradi (per-telefon 60s cooldown himoyada). Qoniqarli.

  faqat raqam ko'rinardi. `onboarding.dart` endi `otpSentTo` l10n kalitini ishlatadi (kalit qo'shilmaguncha
  regressiyasiz fallback — faqat raqam). l10n patch: §7.2.
